## What this is

**Cross-Harness Review** is a **Grok plugin** that lets Grok call **external
coding harnesses and their models** — currently **Claude Code** and **OpenAI
Codex** — to get a read-only **second opinion** on your code, plans, tests, and
security.

Grok stays in the driver's seat: the external reviewer runs in a locked-down,
read-only sandbox, and its output is treated as *untrusted candidate evidence*
that Grok verifies before acting. This is a transport, not an authority.

This is the **first public release** (`v0.1`, npm `0.1.0`).

## Highlights

- 🔌 **Grok plugin** with two skills:
  - `cross-harness-review` — explicit slash entrypoint
    (`/cross-harness-review plan|code|tests|security [scope]`).
  - `cross-harness-auto` — model-invocable; lets Grok auto-run a review from
    natural language ("cross-check with Claude before I open the PR").
- 🧠 **Two external providers** at launch: **Claude Code** (`claude`) and
  **OpenAI Codex** (`codex`), auto-discovered on `PATH`, `%APPDATA%`/npm,
  `%LOCALAPPDATA%`, and WSL; the highest executable semantic version wins.
- 🛡️ **Read-only by construction**: Claude gets no tools (plan) or
  `Read,Grep,Glob` only (code); Codex runs in `read-only` + `--ephemeral` +
  `--ignore-user-config` + `--ignore-rules`. No hooks, agents, active MCP,
  model override, or sandbox-bypass flags.
- 🎯 **Host scope gate**: builds a changed-file allowlist from Git
  (`uncommitted` / `base:<branch>` / `commit:<sha>`) and marks any finding
  whose `evidence.file` is outside the allowlist as `verification: out_of_scope`.
- 🔒 **Bounded execution**: stdin-only prompt delivery, per-run temp directory,
  default 300s timeout, size-capped stdout/stderr/diff.
- 🧪 **Tested** via PowerShell + POSIX fake-CLI matrices (success, quota, auth,
  permission, process failure, invalid output, timeout, out-of-scope gating).

## Install

### From npm

```bash
npm install -g @sidler289-code/cross-harness-review
grok plugin install "$(npm root -g)/@sidler289-code/cross-harness-review" --trust
grok plugin enable cross-harness-review
```

### From GitHub

```bash
grok plugin install https://github.com/sidler289-code/GrokBuild-outsideHarness.git --trust
grok plugin enable cross-harness-review
```

## Requirements

- Grok CLI with plugin support.
- At least one of: **Claude Code CLI** (`claude`) or **Codex CLI** (`codex`),
  installed and authenticated.
- `git` on `PATH` (for the scope snapshot and host scope gate).

## Safety boundary (summary)

- Reviewer text is **never** authority to edit — only the user's request is.
- Unresolvable `base:`/`commit:` scope refs are surfaced explicitly rather
  than silently producing an empty allowlist.
- Claude session JSONL cleanup failure no longer discards an otherwise-valid
  review; it surfaces as a non-fatal `diagnostics.warnings` entry.

See [SECURITY.md](https://github.com/sidler289-code/GrokBuild-outsideHarness/blob/main/SECURITY.md)
for the full boundary and known limitations.

## Known limitations

- Active MCP server is not shipped in v0.1 (optional future work).
- Claude's `--tools ''` (empty string) plan-review semantics are CLI-version
  dependent; the stronger signals are `--permission-mode plan` + `--safe-mode`.
- Providers remain probabilistic; this plugin gives Grok a second opinion, not
  a guarantee.
