## Real-output defect and revision

Two Claude runs produced detailed review JSON but decorated the payload with
Chinese prose and a fenced `json` Markdown block. The Claude adapter correctly
unwrapped the outer `type: result` envelope, but the shared normalizer then ran
`JSON.parse()` against the entire decorated payload and returned
`invalid_output`.

The shared normalizer now tries candidates in this order:

1. the complete stdout string, preserving the strict documented path;
2. `json` or untagged Markdown code fences;
3. balanced outer JSON objects embedded in prose.

Extraction does not bypass validation: a candidate is accepted only if it
satisfies the complete v2 review-result contract for the expected task, role,
and reviewer. The balanced-object scan is linear in output size.

Focused core and Claude-wrapper regression tests pass. The full Node suite
after the revision passes 214/214.

## Codex result

cause. Diagnose it with a separate bounded host-side read-only check before
changing the adapter.
